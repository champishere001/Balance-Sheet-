System Overview

🎯 What is e-SIGMA?

A comprehensive web-based Admin Panel for managing Computer-Assisted Personal Interviews (CAPI) surveys conducted by MoSPI.


The system facilitates real-time survey management, team allocation, task distribution, and progress monitoring for the Periodic Labour Force Survey (PLFS) and other statistical surveys.

🔑 Key Functionalities

  • User Management (Enumerators, Supervisors, ROs)
  • Team Allocation & Task Distribution
  • FSU (First Stage Unit) Assignment
  • Real-time Dashboard & Monitoring
  • Survey Progress Tracking
  • Mobile APK Distribution

Access Control & Authentication

🔐 Secure Login Process

  • Role-based access control (RO User)
  • Unique User IDs (e.g., RO.SML.1)
  • Password protected entry
  • CAPTCHA verification for security
  • Session management

Security Features

Multi-layer authentication ensures only authorized personnel access sensitive survey data and management functions.

Survey Data Flow Process

1️⃣

Assignment Phase

RO allocates FSUs to Supervisors (SSO) → Supervisors assign to Enumerators (JSO) via mobile APK

2️⃣

Data Collection

Enumerators conduct field surveys using mobile tablets → Real-time data entry with validation checks

3️⃣

Submission & Review

Enumerator submits → Supervisor reviews → Returns if errors → Submits to Data Supervisor (DS)

4️⃣

Acceptance & Processing

DS accepts valid data → Data processing → Report generation → Export to statistical databases

Reports & Analytics

📈 Available Reports

  • Detailed Report: FSU-wise comprehensive data
  • Summary Report: Aggregated statistics
  • Referred Back Report: Error tracking
  • Present Status Report: Current state overview
  • Monthly Recap: Periodic summaries

🔍 Filter Capabilities

  • Survey Year selection
  • SCH_0.0 Status filtering
  • FSU Status (All/Specific)
  • Sector (Urban/Rural)
  • Quarter & Month
  • Panel & Visit number
  • Date range selection
  • DS and SRO filtering

Export Functionality

All reports support Excel/CSV export for further analysis. The system shows pagination (e.g., 1 to 10 of 88 entries) for handling large datasets efficiently.
